# Chapter 4: Getting Started

## Core Idea
A GROMACS simulation follows a pipeline: generate topology → define box → solvate → add ions → minimize energy → equilibrate (NVT, then NPT) → production MD → analyze. Understanding the file types is essential: .top (topology), .gro/.pdb (structure), .mdp (parameters), .tpr (run input), .xtc/.trr (trajectory).

## Frameworks Introduced
- **Simulation pipeline**: pdb2gmx → editconf → solvate → genion → grompp (EM) → mdrun (EM) → grompp (NVT) → mdrun (NVT) → grompp (NPT) → mdrun (NPT) → grompp (MD) → mdrun (MD) → analysis
- **Preprocessor include mechanism**: `#include "file.itp"` — cpp-style preprocessing with `-D` defines for conditional topology inclusion

## Key Concepts

### File Types
- **.top**: Molecular topology file generated by `gmx pdb2gmx`; complete description of all interactions
- **.itp**: Include topology file; `#include`'d into .top files. Contains force field parameters
- **.gro**: GROMOS structure file; can hold coordinates and velocities; human-readable fixed-column format
- **.pdb**: Protein Data Bank format; standard input for pdb2gmx, also usable as coordinate format
- **.mdp**: Molecular Dynamics Parameters file; contains ALL simulation settings (integrator, dt, temperature, pressure, cutoffs, output frequency, etc.)
- **.ndx**: Index file; defines groups of atoms for temperature coupling, restraints, analysis
- **.tpr**: Portable run input file; binary combination of .gro + .top + .mdp + .ndx; sole required input for mdrun
- **.trr**: Full-precision trajectory (coordinates, velocities, forces); large but complete
- **.xtc**: Compressed trajectory (coordinates only); smaller, sufficient for most analysis
- **.edr**: Energy data file (binary XDR); read by `gmx energy`
- **.cpt**: Checkpoint file; full-precision state for exact restarts
- **.log**: Text log file from mdrun; contains performance data, warnings, cycle counters
- **.xvg**: Xmgrace plot format; used for all analysis tool output

### Preprocessor Directives
- `#include "file.itp"` — inline include, searched in current dir, GMXLIB, and -I paths
- `#ifdef VAR / #endif` — conditional block; activated by `-D VAR` in mdp define parameter
- `#ifndef / #else` — inverse/alternative conditions

## Code Examples
```bash
# Set up environment
source /usr/local/gromacs/bin/GMXRC

# Generate topology from PDB
gmx pdb2gmx -f protein.pdb -o processed.gro -water spc

# Define simulation box (dodecahedron, 1.0 nm from solute)
gmx editconf -f processed.gro -o box.gro -c -d 1.0 -bt dodecahedron

# Solvate
gmx solvate -cp box.gro -cs spc216.gro -o solvated.gro -p topol.top

# Add ions (neutralize system)
gmx grompp -f ions.mdp -c solvated.gro -p topol.top -o ions.tpr
gmx genion -s ions.tpr -o neutral.gro -p topol.top -pname NA -nname CL -neutral

# Energy minimization
gmx grompp -f em.mdp -c neutral.gro -p topol.top -o em.tpr
gmx mdrun -v -deffnm em
```

## Key Takeaways
1. The .tpr file is the single input mdrun needs — everything is bundled into it
2. Use the `-deffnm` flag with mdrun to use the same prefix for all input/output files
3. The `#include` mechanism prevents duplication and typos in topology files
4. Set `GMXLIB` environment variable to point to your custom force field directories
5. Always check the .log file after mdrun for performance data and warnings
